# 🤖 pikabu-bot-v3

Telegram-бот для парсинга и доставки контента с Pikabu.ru. Подписки по тегам, авторам и сообществам — всё через удобное меню прямо в боте.

## Возможности

- **🖼 Картинки по тегам** — создаёте наборы тегов, бот присылает только изображения из подходящих постов (посты без картинок пропускаются)
- **📰 Посты по тегам** — полные посты (заголовок, текст, картинки) по выбранным тегам
- **👤 Подписка на авторов** — все новые посты от выбранных авторов Pikabu
- **👥 Подписка на сообщества** — новые посты из отслеживаемых сообществ
- **Гибкая фильтрация** — в каждом наборе тегов есть включения и исключения
- **Автоматический парсинг** — бот сам проверяет новые посты каждые 10 минут
- **Дедупликация** — один и тот же пост не отправляется дважды
- **Админ-панель** — статистика, ручной запуск парсинга

## Стек

| Компонент | Технология |
|-----------|-----------|
| Runtime | [Bun](https://bun.sh/) |
| База данных | [sql.js](https://github.com/sql-js/sql.js/) (SQLite, чистый JS — без нативных зависимостей) |
| Парсинг HTML | [cheerio](https://cheerio.js.org/) |
| Telegram API | [node-telegram-bot-api](https://github.com/yagop/node-telegram-bot-api) |
| Контейнеризация | Docker + docker-compose |

## Архитектура

```
src/
├── main.ts            — точка входа, инициализация БД и бота
├── telegram-bot.ts    — обработчики команд, меню, рассылка постов
├── pikabu-parser.ts   — HTTP-парсер Pikabu (windows-1251, rate-limiting)
└── storage.ts         — sql.js: пользователи, подписки, теги, статистика
```

База данных хранится в файле `data/bot.db`, автоматически сохраняется на диск каждые 30 секунд. При перезапуске контейнера данные сохраняются через Docker volume.

## Как работает парсер

1. Собирает все активные теги, авторов и сообщества от пользователей
2. По очереди запрашивает страницы Pikabu с задержкой 2–4 секунды между запросами (защита от rate-limit)
3. Ответы Pikabu приходят в windows-1251 — бот декодирует вручную через таблицу символов
4. Извлекает посты: заголовок, автор, теги, изображения, текст, рейтинг
5. Фильтрует по подпискам каждого пользователя, проверяет дедупликацию
6. Отправляет подходящие посты в Telegram

## Запуск

### Через Docker (рекомендуется)

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/sakurka-cmd/pikabu-bot-v3.git
   cd pikabu-bot-v3
   ```

2. Создайте `.env` файл:
   ```
   BOT_TOKEN=ваш_токен_от_BotFather
   ```

3. Запустите:
   ```bash
   docker-compose up -d
   ```

Бот автоматически стартует и начинает парсинг каждые 10 минут.

### Локально (для разработки)

```bash
bun install
BOT_TOKEN=ваш_токен bun run src/main.ts
```

## Команды бота

Бот работает через меню клавиатуры — команды `/start` и `/menu` доступны всегда:

- `/start` — запуск бота и регистрация
- `/menu` — вернуться в главное меню

Главное меню:
| Кнопка | Описание |
|--------|----------|
| 🖼 Картинки | Наборы тегов для получения только изображений |
| 📰 Посты | Наборы тегов для получения полных постов |
| 👤 Авторы | Подписки на авторов Pikabu |
| 👥 Сообщества | Подписки на сообщества |
| 📊 Статистика | Персональная статистика |
| ⚙️ Админ | Панель администратора (только для первого пользователя) |

## Ограничения

- Pikabu может отдавать 403/429 — парсер автоматически ждёт при rate-limit (60 сек)
- Доступ к 18+ контенту не гарантирован (зависит от политики Pikabu)
- Максимальное количество наборов, тегов и подписок настраивается в настройках бота
